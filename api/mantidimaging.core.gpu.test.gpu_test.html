class mantidimaging.core.gpu.test.gpu_test.GPUTest(*args, **kwargs)[source]¶

Bases: unittest.case.TestCase

Test median filter.

Tests return value and in-place modified data.

static run_serial(data, size, mode)[source]¶

Run the median filter in serial.

test_array_input_unchanged_when_gpu_runs_out_of_memory()[source]¶

Mock the GPU running out of memory. Check that this leaves the input array to be unchanged.

test_double_is_used_in_cuda_for_float_64_arrays()[source]¶

Run the median filter on the CPU and GPU with a float64 array. This demonstrates that replacing instances of ‘float’ with ‘double’ in the CUDA file is doing the right thing.

test_gpu_result_matches_cpu_result_for_different_filter_sizes()[source]¶

Run the median filter on the CPU and GPU with different filter sizes. Check that the results match.

test_gpu_result_matches_cpu_result_for_larger_images()[source]¶

Run the median filter on the CPU and GPU with a larger image size. Check that the results match. This test may reveal issues such as the grid and dimension size arguments going wrong.

test_gpu_running_out_of_memory_causes_free_memory_to_be_called()[source]¶

Mock the GPU running out of memory. Check that this causes the free memory block function to be called.

test_image_slicing_works()[source]¶

Run the median filter on the CPU and GPU with an image stack that is larger than the limit permitted on the GPU. This demonstrates that the algorithm for slicing the stack and overwriting GPU arrays is working correctly.

test_numpy_pad_modes_match_scipy_median_modes()[source]¶

Run the median filter on the GPU and CPU with the different scipy modes. Check that the results match. Should demonstrate that the arguments passed to numpy pad are the correct equivalents to the scipy modes.
